üìù 365WineTrade Notes: Wine Industry Item Master Data

365WineTrade adds extensive wine-specific fields to Item Cards:

  • 365WineTrade FastTab - Brand, Producer, Vintage, Varietal, Label, Alcohol Type, Wine Color, Origin Region, Appellation
  • Case Pack field - automatically creates unit of measure conversions between bottles and cases
  • Unit Sizes - standardized bottle sizes (750ml, 1.5L, etc.)
  • Item Reviews - track ratings from publications (Wine Spectator, etc.) with scores and review dates
  • Enhanced Copy Item function - copy items across different companies in your database

Wine-specific master data tables you'll work with:

  • Brands and Producers (with optional dimension integration)
  • Vintages (including "NV" for non-vintage products)
  • Varietals, Labels, Appellations, Sub-Regions

These fields enable reporting by brand, vintage, and varietal - critical for wine industry analytics.

üìù 365WineTrade Notes: Enhanced Purchase Order Features

365WineTrade extends purchase order functionality:

  • Blanket Purchase Order Synchronization - system can automatically link purchase orders to existing blanket orders, with configurable warnings
  • Purchase Order Consolidations - group multiple purchase orders into consolidations for tracking (domestic or international shipments)
  • Consolidation tracking includes: Container No., Vessel No., Port of Departure, Port of Entry, Expected Receipt Date
  • Batch Post Consolidations - receive and invoice all orders in a consolidation at once
  • Pickup Address field - specify intermediate pickup locations for shipments
  • Freight Forwarder code - track shipping agents on purchase orders

Consolidations help manage container shipments and provide visibility into in-transit inventory.

üìù 365WineTrade Notes: Sales Order Processing Differences

365WineTrade significantly changes sales order processing:

Required Fields:

  • Order Type Code - determines location, charges, and approval requirements (required before entering lines)
  • Sales Region Code - determines which items can be sold (defaults from customer, required)

Sales Order Charges (automatic):

  • Minimum Order Charges - if order doesn't meet minimum bottles/cases/dollars
  • Split Case Fees - charged per bottle outside a full case pack (or per order)
  • Charges are linked to Order Types and triggered automatically

Sales Order Reviews (approval workflow):

  • When you Release an order, it may require approval before it becomes truly "Released"
  • Status shows "Open" with Review Status "Pending" until approved
  • Review types include: Credit holds, Sample orders, Price changes, Non-reserved quantities, etc.
  • Orders appear on Sales Order Review Dashboard or Credit Review Dashboard for approval

Other enhancements:

  • Delivery Instructions field (flows from customer/ship-to address)
  • Pickup Address as ship-to option
  • Credit Class Code (triggers credit reviews when on hold)

Sales order processing will feel significantly different from standard BC due to these approval workflows.

üìù 365WineTrade Notes: Advanced Wine Industry Pricing

365WineTrade adds powerful pricing models beyond standard BC:

Mix & Match Pricing:

  • Consolidates quantities across multiple items with the same Sales Price Group
  • System finds best price based on total consolidated quantity
  • Each price list line can be enabled/disabled for Mix & Match

Co-op Pricing:

  • For New Jersey Co-ops (or similar buying groups)
  • Consolidates quantities across ALL orders for the same co-op and same date
  • Ensures all co-op members get the best case price
  • Each price list line can be enabled/disabled for Co-op

Family Plans Pricing:

  • Finds best price for each item based on the total case count of the entire order
  • Each price list line can be enabled/disabled for Family Plans

Customer Specific Pricing:

  • Overrides BC's "best price" behavior - if you set a customer-specific price, that price is used even if a lower price exists

Pricing Configuration:

  • Price Starting Date - prices can be based on Shipment Date (not just Order Date)
  • Pricing calculations happen when the order is Released

These pricing models are configured on Customer Price Groups and enabled per price list line.
